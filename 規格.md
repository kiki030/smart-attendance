## 專題名稱：校園智慧點名系統

* **開發原則：** 採用免費、開源框架與演算法模型，確保系統的可維護性與低部署成本。  
* **去識別化特徵儲存**：本系統嚴格遵循個資保護原則，後端資料庫僅儲存去識別化之 512 維浮點數特徵向量（Embedding），不留存任何原始影像或相片檔案，從根本上杜絕人臉隱私外洩風險。 

---

**一、 技術棧選型與建構工具** 

### **1.後端與 AI 引擎 (Backend & AI Engine)**

* **核心開發語言：** Python 3.10+  
* **Web 服務框架：**Python FastAPI  
  * **選型理由：** 內建支援 **OpenAPI (Swagger) 規範**，可自動生成標準化 API 接口文件，利於團隊協作；具備高並發的非同步（Async/Await）I/O 處理能力，適合處理影像串流與大量簽到請求。  
* **人臉偵測（Face Detection）演算法：** YOLOv8-Face  
  * **選型理由：** 屬於目前開源界主流的單階段（Single-stage）物件偵測架構。相較於傳統 RetinaFace，YOLOv8-Face 在保持極高偵測率的同時，具備更低的推論延遲（Low Inference Latency），能實時且精準地在教室複雜場景中圈選出多個人臉框。  
* **人臉識別（Face Recognition）演算法：** ArcFace (InsightFace)  
  * **選型理由：** 完全開源且具備頂尖的學術代表性。其採用加性角度餘弦損失函數（Additive Angular Margin Loss），在數學原理上極大地拉開了類間距離並縮小類內距離。對於教室場景中常見的**側臉、低頭、光線變化**，ArcFace 的 512 維度特徵向量（Embedding）具備極高的魯棒性（Robustness）。

### **2\. 前端使用者介面 (Frontend View)**

* **核心開發框架：** React.js  
* **樣式與 UI 框架：** Tailwind CSS  
* **選型理由：**  
  * **響應式網頁設計 (RWD)：** 確保單一程式碼庫即可完美適配桌上型電腦（教師端操作）與智慧型手機（師生行動端查看）。  
  * **前後端分離：** 前端僅透過 Axios 非同步呼叫後端 API，符合現代企業級軟體工程標準，與後端解耦，技術深度符合資工系要求。

### **3\. 資料庫與資料結構設計 (Database & Storage)**

* **關聯式資料庫 (RDBMS)：** MySQL  
  * **選型理由：** 開源且免費。用於儲存具備強關聯性的結構化資料，系統將善用其**外鍵約束 (Foreign Key Constraints)** 確保資料完整性，並規劃撰寫**預存程序 (Stored Procedure)** 進行後台數據統計。  
* **特徵向量檢索方案：** Python 記憶體矩陣運算 (NumPy Array Cache)  
  * **選型理由（架構創新點）：** 考量學校班級人數規模（百人至千人級別），系統於啟動時會將 MySQL 中的學生 512 維人臉特徵向量預先載入至記憶體中。當鏡頭捕捉到人臉時，直接利用 NumPy 進行高速的**餘弦相似度 (Cosine Similarity)** 矩陣運算，免去逐筆查詢資料庫的 I/O 損耗，達成毫秒級的實時比對。

## **二、 系統核心功能模組 (Core Modules)**

* **影像預處理模組：** 透過 OpenCV 對 USB 鏡頭串流進行幀率優化抽樣（如 1 秒 2 幀），並導入 CLAHE 演算法調整對比度，對抗教室投影機與窗外光線變化。  
* **多人實時辨識模組：** YOLOv8-Face 多人臉並行偵測 影像裁切對齊 ArcFace 特徵提取 NumPy 記憶體比對。  
* **防偽與權重控制模組：** 採用多幀投票機制（連續多幀判定為同一人始完成簽到），並規劃加入輕量級活體檢測（如眨眼偵測），防止照片或影片作弊。  
* **視覺化管理後台：** 提供教師端一鍵匯出簽到報表、動態缺席率圖表（利用 React 圖表庫），以及手動補點名之權限控管。  
* **例外處理與防呆模組：　**  
  * **未知人臉過濾 ：**設定餘弦相似度閥值（Threshold），凡低於閥值之辨識結果統一歸類為 \`Unknown\`，並於教師端即時流中以紅色框提示，但不寫入 MySQL 正常出勤表，防止誤判。   
  * **斷線重連機制 ：**當鏡頭串流中斷或網絡延遲時，OpenCV 模組需具備自動嘗試重連（Auto-reconnect）機制。

**三、設計規格：**

  * 角色切換開關（教師 / 學生）。  
    * 學號/工號與密碼輸入框。  
    * *手機端優化*：支援記住帳號與密碼，方便快速登入。

  * ### **2\. 教師主控台 (Teacher Dashboard) \- 核心畫面**

    * **課程列表**：顯示當天或本週該教師要上的課。  
    * 增加「啟動課堂點名」與「關閉點名結算」之功能按鈕，用以控管後端影像抽樣與 AI 推論之生命週期，避免伺服器資源浪費。   
    * **即時點名監控面板（重點）**：  
      * **左側（或上方）**：USB 鏡頭即時影像串流（顯示 YOLOv8 畫出的人臉框與辨識名字）。  
      * **右側（或下方）**：即時出勤統計圖表（已到人數、遲到人數、未到人數）。

  * **4\. 學生人臉註冊頁面 (Face Registration View)**   
    * **引導式 UI**：指引學生使用手機/電腦鏡頭進行多角度（正面、側面、俯仰）人臉數據採集，前端利用 Canvas 進行人臉裁切預覽，確保上傳品質。 

**四、 專案版本控制** 

本專案之軟體工程生命週期嚴格遵循 **Git 工作流（Git Workflow）** 規範，確保程式碼之可追溯性與團隊協作安全：

* **儲存庫架構 (Repository Layout)：** 採用單一儲存庫（Mono-repo）形式整合前後端系統。  
* **分支管理策略 (Branching Strategy)：**  
  * `main` 分支：僅存放經嚴格測試、可穩定運作之 Production 生產版本，作為最終成果展示之基準。  
  * `dev` 分支：日常功能開發與 AI 模型調校之主分支。  
* **自動化版本控制：** 於 Antigravity 開發環境中整合 Git 代理，依功能模組（如 UI 切版、API 串接）進行具備語義化（Semantic Commits）的階段性提交。

**五、 成果展示與雙平台部署方案**

* **前端 Web 靜態託管 (Vercel Platform)：**  
  * 將 frontend 目錄透過 CI/CD 機制連動至 **Vercel** 免費雲端平台。每當程式碼合併至 main 分支，系統將自動觸發 Vite 編譯生產環境檔案並完成自動部署。  
  * **預期產出：** 產生一組公開之 HTTPS 網址（例如：https://smart-attendance.vercel.app），供教師與學生從任何智慧型手機或電腦點擊登入。  
* **後端與 AI 運算端穿透 (本地主機 \+ ngrok 內網穿透)：**  
  * 考量 YOLOv8-Face 與 ArcFace 模型在推論時對硬體運算資源（CPU/RAM）之高需求 ，後端 FastAPI 服務與 USB 鏡頭採集端將運行於本地開發主機 。  
  * 本地端啟動 **ngrok 反向代理工具**，將本地端埠號（Port）安全地對外映射為一組臨時的全球 HTTPS 網址（例如：https://xxxx.ngrok-free.app）。  
* **系統動態串接驗證：**  
  * 將 Vercel 前端之環境變數 API 節點（API\_BASE\_URL）動態指向該 ngrok 網址。評審教授於外部網路點擊前端網址時，前端即可跨網域（CORS）調用本地主機的 AI 核心，實時讀取課堂鏡頭畫面並寫入資料庫，達成完整系統的端到端（End-to-End）實機 DEMO。
